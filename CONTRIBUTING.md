# Contributing to GigMarket

Thank you for your interest in contributing to GigMarket! We welcome contributions from developers, builders, and researchers.

## 🚀 How to Get Started

1. **Fork the Repository**: Create a personal fork on GitHub.
2. **Clone Locally**: 
   ```bash
   git clone https://github.com/chaayadev1995/gig-market.git
   cd gig-market
   ```
3. **Set Up the Environment**:
   - Copy `.env.example` to `.env`
   - Fill in your local/test variables
4. **Install Dependencies**:
   ```bash
   npm install
   ```
5. **Create a Feature Branch**:
   ```bash
   git checkout -b feat/your-feature-name
   ```

## 🛠️ Code Conventions & Guidelines

* **Clean Commits**: Commit logic in granular, readable groups with clear commit messages.
* **Solidity**: Ensure your Solidity contracts compile cleanly using `npm run compile` and match the designated Solidity `0.8.20` compiler settings.
* **Testing**: Run local tests and verify on the Arc Testnet before submitting pull requests.

## 📥 Submitting Pull Requests

1. Commit your changes and push them to your fork.
2. Open a Pull Request (PR) targeting the `main` branch.
3. Describe the changes, issues solved, and test verification details in the PR description.
